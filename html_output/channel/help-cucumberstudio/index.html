So you can simple add --parameter-delimiter="" each time you have to execute the hiptest-publisher command lines to update your features. For example:

--> To update your tests: hiptest-publisher -c &lt;path to your config file&gt; --only=features,step_definitions --parameter-delimiter=""

--> To regenerate your tests and fetch UIDs for each scenarios included in your test run:

hiptest-publisher --config-file &lt;path to your config file&gt; --test-run-id &lt;your test run ID&gt; --without=actionwords --parameter-delimiter=""

*Thread Reply:* Hi @Michael Stelly,

There are historically two kind of projects: BDD with Cucumber, and Test Management. Test Management mimics what Hiptest was doing before we rebranded it as CucumberStudio, and BDD with Cucumber is where we developed features which better match Cucumber users expectations.

We recently realised it was more confusing than useful to have two kinds of projects. All the features provided by the BDD with Cucumber project type have been migrated to the Test Management project type. We’ll retire the BDD with Cucumber project type in the coming weeks.

So better use Test Management project type.

About Discovery feeding Formulation, I totally agree. You can link your Jira tickets to user stories in CucumberStudio, create Example Maps for these user stories. That’s the Discovery part. You can then use these example maps to update or create features and scenarios. That’s the Formulation part.

For now, there is now easy way to use the example map content to create the scenarios. That’s something we are working on.

  2. Get tests of a test run, to fetch test snapshots IDs: https://studio-api.cucumber.io/#get-tests-of-a-test-run
  3. Add a test execution result: https://studio-api.cucumber.io/#add-a-test-execution-result you can also update test results with: https://studio-api.cucumber.io/#update-a-test-execution-result

And this can be applied to a test run which only has one default execution environment.

In case your test run has multiple execution environments: • Create a new build in a specific execution environment: https://studio-api.cucumber.io/#create-a-new-build-in-a-specific-execution-environment • Get tests of a test run, to fetch test snapshots IDs: https://studio-api.cucumber.io/#get-tests-of-a-test-run • Add a test execution result to the current build of an execution environment: https://studio-api.cucumber.io/#add-a-test-execution-result-to-the-current-build-of-an-execution-environment
