Will declaring or initializing WebDriver as static will it effect parallel execution ?

Automate_Tests
2019-09-20 19:35:08

*Thread Reply:* it depends on how/where you initialize it, but yeah, it'll probably cause problems

Automate_Tests
2019-09-20 19:42:32

*Thread Reply:* ideally, you want one instance of webdriver, that you can pass from method to method, and making it static means you'll either have multiple tests accessing the same webdriver instance, or each step will be using it's own webdriver instance (neither will give you the results you want)

delso
2019-09-21 15:22:52

*Thread Reply:* Would synced be better?

Jeevan L
2019-10-10 03:24:30

*Thread Reply:* thanks @Automate_Tests

Automate_Tests
2019-10-10 05:39:50

*Thread Reply:* There are a few ways around this, typically, I'll do one of two things. 1) Use a Before method to declare a new WebDriver object, and pass it into each of my test steps 2) Use dependency injection to pass and share the WebDriver instance through each test step class. This works best if you setup your own implementation, which can determine what type of WebDriver to declare (Chrome, Firefox, etc), which allows a lot of flexibility
