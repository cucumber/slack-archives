aslakhellesoy (https://github.com/aslakhellesoy)
#712 Use protobuf instead of ndjson in gherkin acceptance tests
<p>It's currently possible to pass the Gherkin acceptance test suite without implementing support for cucumber-messages (protobuf).</p> <p>By making the test suite compare binary protobuf output instead of ndjson we'd be forcing implementations to implement cucumber-messages.</p> <p>This may sound a little strict, but I think it is the right thing to do in order to achieve better platform compatibility and interop with e.g. standalone formatters such as the new <a href="https://github.com/cucumber/cucumber/tree/master/html-formatter">html-formatter</a> and <a href="https://github.com/cucumber/cucumber/tree/master/json-formatter">json-formatter</a>.</p> <p>I propose we start a <code>gherkin-protobuf-tests</code> branch where we collaborate on bringing gherkin-dotnet and gherkin-perl uptodate. gherkin-java, gherkin-javascript and gherkin-ruby already are.</p> <p>WDYT <a href="https://github.com/ehuelsmann">@ehuelsmann</a> <a href="https://github.com/gasparnagy">@gasparnagy</a> <a href="https://github.com/SabotageAndi">@SabotageAndi</a> ?</p>
