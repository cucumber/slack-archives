2020-06-16 01:21:25

Just to let everyone know, the lwt implementation has stalled slightly. I am struggling with a way to wrap the monads in such a way that I can capture the outcome of each and only pass the state forward to the next computation. Honestly, I think that passing the list of outcomes and the current state should be OK because if someone wants to change the history of the outcomes of their previous steps, they can do that any number of other ways anyway so it might not be too interesting to just pass forward the current list. It makes it easier if I cannot figure out how to capture it.
