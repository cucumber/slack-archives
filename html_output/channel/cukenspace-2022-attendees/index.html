*Thread Reply:* Overall, yes, this was exactly what we talked about 👍 You’re catching the exception just in case, and then testing that you had it thrown when it should fail 💯

Some quick thoughts on the Gherkin: • I usually frown on multiple “When” steps. You want the action that you’re taking on the system to be clear when reading. • Always think Context( Givens) + Action (When) = Outcome (Thens) • If you’re taking multiple actions, then it can be confusing which one is really doing the work. • Of course this is all syntactically equivalent. The code will run the same. But when expressing it in Gherkin, I want to be super clear what action I’m taking and what outcome I expect that action to have. So that might change the gherkin slightly: ```Given Sally has a valid one-time code When she registers using that code Then she should be logged in
