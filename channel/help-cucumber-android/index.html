davidvavra (https://github.com/davidvavra)
#331 Support for pure Kotlin tests (needed for Kotlin Multiplatform)
<p>Kotlin now has support for multiplatform projects. Common code is written in pure Kotlin (no Java dependencies are possible) and then developer creates modules like <code>common-jvm</code> and <code>common-js</code> which can contain JVM or JS dependencies. The code in the common module should be tested and Jetbrains created annotations in <code>kotlin.test</code> package which are automatically translated into JUnit or JS tests. Then tests can run on all platforms with no work. <a href="https://kotlinlang.org/docs/reference/multiplatform.html#multiplatform-tests">More documentation here</a>.</p> <p>So currently cucumber can be used only in <code>common-jvm</code> or <code>common-js</code> modules, but not in the common module. Which means writing step definitions twice. It would be great if cucumber supported <code>kotlin.test</code> tests directly.</p>
