Bartosz Żołyński
2023-11-13 02:27:46

*Thread Reply:* The problem might be also that the technical implementation gets slightly hairy. I didn’t make it working with a transpiling magic for React application as I wanted to make the living documentation on the unit-test level, so I used Cypress, but this is sort of E2E.

In order to have different edge cases, I used mocking of some API responses, but this is already something disliked. The way it’s done is not straightforward, as API calls are server-side so intercept from Cypress cannot be used. You have to pass something from the browser (client-side) to the server-side (express server in this case) so then the environment “knows” that it should return something specific. To do it, I used to send query parameters that are received and stored in-memory on server-side so once express server (server-side) requests API call, if parameter exists in that storage, it knows to return a mock instead of doing a real call to external API.

This way I can test different cases as I can manipulate the response. I’ve found this POC really fun to work with, as I can easily reproduce different states of the application with just these query parameters. It’s convenient to reproduce state when developing something new. And this also can be part of the documentation so you can see it as a storybook! But it feels controversial to some of the developers... So… it’s hard for me to convince about the benefits of my idea with that Decoupled Context as I call it.
